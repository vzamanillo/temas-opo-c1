# MODELOS DE RED: OSI Y TCP/IP

## 1. EL MODELO DE REFERENCIA OSI

Creado por la ISO, divide la comunicación en 7 capas para estandarizar el proceso. [1]

* Capas de Aplicación (Usuario):
  * 7 Aplicación: Interfaz con el usuario (HTTP, FTP, SMTP).
  * 6 Presentación: Formato y cifrado de datos (JPEG, ASCII, TLS/SSL).
  * 5 Sesión: Mantenimiento de la conexión entre aplicaciones.
* Capas de Transporte y Red (Corazón):
  * 4 Transporte: Entrega extremo a extremo y control de errores (TCP, UDP). Aquí hablamos de Segmentos.
  * 3 Red: Direccionamiento lógico y determinación de ruta (IP, ICMP). Aquí hablamos de Paquetes.
* Capas Físicas (Hardware):
  * 2 Enlace de datos: Direccionamiento físico (MAC, Ethernet). Aquí hablamos de Tramas.
  * 1 Física: Transmisión de bits por el medio (cables, ondas de radio). [2, 3, 4, 5, 6]

## 2. EL MODELO TCP/IP

Es el modelo práctico que hace funcionar Internet. Es más sencillo y agrupa las capas de OSI en 4 niveles: [7, 8, 9, 10]

   1. Capa de Aplicación: (Agrupa las capas 5, 6 y 7 de OSI). Define los protocolos de los servicios (Navegación, correo, etc.).
   2. Capa de Transporte: (Capa 4 de OSI). Gestiona la comunicación host a host.
   3. Capa de Internet: (Capa 3 de OSI). Encargada de que los paquetes lleguen a su destino a través de diferentes redes.
   4. Capa de Acceso a la Red: (Agrupa las capas 1 y 2 de OSI). Define cómo se transmiten los datos físicamente. [11, 12, 13, 14, 15]

## 3. COMPARATIVA Y CORRESPONDENCIA

| Capa OSI [16, 17, 18, 19, 20] | Capa TCP/IP | Unidad de Datos (PDU) | Protocolos Ejemplos |
| --- | --- | --- | --- |
| 7. Aplicación | Aplicación | Datos | HTTP, DNS, SMTP |
| 6. Presentación | Aplicación | Datos | SSL, TLS, JPG |
| 5. Sesión | Aplicación | Datos | RPC, NetBIOS |
| 4. Transporte | Transporte | Segmento | TCP, UDP |
| 3. Red | Internet | Paquete | IP, ICMP, ARP |
| 2. Enlace | Acceso a Red | Trama | Ethernet, Wi-Fi |
| 1. Física | Acceso a Red | Bit | Cable cobre, Fibra |

## 4. CONCEPTOS CLAVE

* Encapsulamiento: Proceso por el cual, al bajar de capa, se le añade información de control (cabecera) a los datos. Al recibir, se realiza el proceso inverso (desencapsulamiento).
* Interfaz: Es el punto de comunicación entre dos capas adyacentes en el mismo sistema (ej. cómo la capa de Red le entrega datos a la de Transporte).
* Direccionamiento:
  * En Capa 2 (Enlace) usamos la Dirección MAC (física, grabada en la tarjeta).
  * En Capa 3 (Red/Internet) usamos la Dirección IP (lógica).
  * En Capa 4 (Transporte) usamos los Puertos (identifican la aplicación). [21, 22, 23, 24, 25]

Dato clave: Los Routers son dispositivos de Capa 3 (trabajan con IPs), mientras que los Switches tradicionales son de Capa 2 (trabajan con direcciones MAC). [26, 27]
