# VIRTUALIZACIÓN DE SISTEMAS

## 1. CONCEPTO DE VIRTUALIZACIÓN

Es la tecnología que permite crear una capa de abstracción sobre el hardware físico, dividiendo sus recursos (CPU, RAM, Disco) en múltiples entornos virtuales independientes y aislados.

## 2. EL HIPERVISOR (ARQUITECTURAS)

El Hipervisor o VMM (Virtual Machine Monitor) es el software que gestiona el hardware y reparte los recursos entre las máquinas virtuales. Existen dos tipos:

* Tipo 1 (Bare Metal): El hipervisor se instala directamente sobre el hardware físico. Es el más eficiente y usado en CPDs.
  * Ejemplos: VMware ESXi, Microsoft Hyper-V, Citrix Hypervisor.
* Tipo 2 (Hosted): El hipervisor corre como una aplicación sobre un sistema operativo ya instalado (Windows, Linux). Se usa para pruebas o desarrollo.
  * Ejemplos: VirtualBox, VMware Workstation.

## 3. MÁQUINAS VIRTUALES (VM)

Es un contenedor de software que incluye un Sistema Operativo Invitado (Guest OS) y sus aplicaciones. Para el hardware físico, la VM es solo un proceso; para el usuario, funciona exactamente igual que un ordenador real.

* Aislamiento: Si una VM falla o se infecta con un virus, no afecta a las demás ni al servidor físico.

## 4. VIRTUALIZACIÓN DE SERVIDORES

Es la aplicación de esta tecnología en el ámbito profesional para consolidar servidores.

* Escenarios de aplicación:
  * Consolidación: Pasar de 10 servidores físicos infrautilizados a 1 solo servidor potente con 10 VMs.
  * Entornos de prueba: Crear y destruir servidores rápidamente sin coste de hardware.
  * Alta Disponibilidad: Mover una VM de un servidor físico a otro sin que el servicio se detenga (Live Migration).
  * Legado: Ejecutar aplicaciones antiguas en sistemas operativos obsoletos sobre hardware moderno.

## 5. GESTIÓN Y MONITORIZACIÓN

Para administrar estos entornos se necesitan consolas centralizadas que permitan:

* Provisionamiento: Crear nuevas VMs en minutos.
* Balanceo de carga: Mover VMs automáticamente al servidor físico que tenga menos carga (ej. VMware DRS).
* Monitorización: Controlar el consumo de recursos para evitar la "sobreasignación" (dar más RAM de la necesaria a una VM).
* Snapshots: Capturar el estado de una VM antes de un cambio crítico para poder volver atrás al instante si algo sale mal.

## 6. SOLUCIONES DE MERCADO

* VMware vSphere: El estándar de oro en empresas (Hipervisor ESXi).
* Microsoft Hyper-V: Integrado en Windows Server, muy popular en entornos Windows.
* KVM (Kernel-based Virtual Machine): La solución libre líder en Linux (base de muchas nubes).
* Proxmox VE: Solución de código abierto basada en KVM muy popular por su flexibilidad.

Dato clave: La virtualización reduce drásticamente el consumo de energía y espacio en el CPD (ahorro de OPEX y CAPEX).
