# SERVICIOS Y PROTOCOLO DE RED

## 1. SERVICIOS BÁSICOS EN RED

Son las aplicaciones que permiten al usuario realizar tareas sobre la infraestructura de Internet:

* Resolución de nombres (DNS): Traduce nombres de dominio (ej. google.es) en direcciones IP numéricas que las máquinas entienden. Sin DNS, tendríamos que recordar las IPs de cada sitio web.
* Correo Electrónico: Sistema de intercambio de mensajes. Utiliza una arquitectura de "almacenamiento y reenvío" mediante servidores de correo.
* Transferencia de ficheros (FTP): Servicio dedicado al envío y recepción de archivos entre un cliente y un servidor.
* Acceso Remoto: Permite controlar un ordenador desde otro lugar (ej. SSH, RDP).
* Web (HTTP/HTTPS): Sistema de distribución de documentos de hipertexto o multimedia interconectados y accesibles vía Internet. [1, 2, 3, 4, 5]

## 2. PROTOCOLOS DE CORREO ELECTRÓNICO

Existen tres protocolos fundamentales que trabajan juntos: [6]

* SMTP (Simple Mail Transfer Protocol): Se usa para enviar correos (desde el cliente al servidor o entre servidores).
* POP3 (Post Office Protocol): Se usa para descargar correos del servidor al dispositivo local. Normalmente los borra del servidor tras descargarlos.
* IMAP (Internet Message Access Protocol): Se usa para consultar correos. Los mensajes permanecen en el servidor y se sincronizan entre múltiples dispositivos (móvil, PC, web). [7, 8, 9, 10, 11]

## 3. PROTOCOLOS DE LA WEB Y NAVEGACIÓN

La navegación web se basa en el modelo petición-respuesta: [12, 13]

* HTTP (HyperText Transfer Protocol): Protocolo "sin estado" (cada petición es independiente) que envía los datos en texto plano. Es vulnerable porque cualquiera que intercepte la red puede leer la información.
* HTTPS (HTTP Secure): Es la versión segura de HTTP. Utiliza una capa de cifrado para que los datos viajen protegidos.
* SSL / TLS: Son los protocolos de seguridad que hay "debajo" del HTTPS:
  * SSL (Secure Sockets Layer): El protocolo original (hoy considerado inseguro y obsoleto).
  * TLS (Transport Layer Security): El sucesor moderno y seguro de SSL. Proporciona:
    * Cifrado: Para que nadie lea los datos.
    * Integridad: Para que los datos no se alteren en el camino.
    * Autenticación: Mediante Certificados Digitales que aseguran que el sitio web es quien dice ser. [14, 15, 16, 17, 18]

## 4. OTROS PROTOCOLOS CLAVE EN INTERNET

* TCP (Transmission Control Protocol): Garantiza que los datos lleguen sin errores y en orden (orientado a conexión).
* UDP (User Datagram Protocol): Más rápido pero no garantiza la llegada (usado en streaming o juegos online).
* IP (Internet Protocol): Se encarga de direccionar y encaminar los paquetes de datos de origen a destino. [19, 20, 21, 22, 23]

Dato clave: Cuando ves un "candado" en el navegador, significa que estás usando HTTPS sobre TLS, lo que garantiza que tu contraseña o tarjeta de crédito viaja cifrada desde tu PC hasta el servidor. [24, 25, 26]

Aquí tienes la explicación paso a paso de cómo funciona la resolución de una consulta DNS, desde que escribes la dirección en el navegador hasta que tu equipo obtiene la IP. Es lo que se conoce como una consulta recursiva:

## El Proceso de Resolución DNS paso a paso

   1. Cache Local: Antes de salir a la red, tu ordenador mira en su propia memoria (caché) y en el archivo hosts. Si ya ha visitado esa web recientemente, resuelve la IP ahí mismo.
   2. DNS Resolver (Recursivo): Si no está en caché, la petición va al servidor DNS de tu proveedor de Internet (ISP). Él será el encargado de "preguntar" a los demás.
   3. Root Nameservers (Servidores Raíz): El resolver pregunta a los servidores raíz (hay 13 principales en el mundo). Estos no saben la IP, pero saben quién gestiona el sufijo (ej. el .es).
   4. TLD Nameservers (Top-Level Domain): El servidor raíz redirige al resolver hacia el servidor del TLD correspondiente (el de los dominios .es). Este sabe quién es el registrador específico del dominio.
   5. Authoritative Nameserver (Servidor Autoritativo): Finalmente, el resolver llega al servidor que tiene la "verdad oficial" sobre el dominio (ej. los DNS de google.es). Este le entrega la Dirección IP exacta (ej. 142.250.184.3).
   6. Respuesta al cliente: El DNS del ISP guarda esa IP en su caché para la próxima vez y se la entrega a tu navegador.
   7. Conexión: Tu navegador ya puede abrir la conexión HTTP/HTTPS directamente contra esa dirección IP.
