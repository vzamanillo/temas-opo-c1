# ESTRUCTURAS DE DATOS Y ORGANIZACIÓN DE FICHEROS

## 1. CONCEPTO DE TIPO DE DATOS

Un tipo de dato es un atributo que indica al ordenador (y al programador) sobre qué clase de datos se va a trabajar. Esto define:

* El espacio en memoria que ocupará (ej. un integer suele ocupar 4 bytes).
* El rango de valores posibles.
* Las operaciones que se pueden realizar con ellos (no puedes "sumar" dos letras de la misma forma que dos números).

## 2. ESTRUCTURAS FUNDAMENTALES DE DATOS

Se dividen principalmente en dos categorías según cómo gestionan la memoria:

## A. Estructuras Estáticas (Tamaño fijo)

* Arrays (Vectores/Matrices): Colección de elementos del mismo tipo almacenados en posiciones contiguas de memoria. Su acceso es muy rápido mediante un índice.

## B. Estructuras Dinámicas (Tamaño variable)

* Listas Enlazadas: Elementos (nodos) donde cada uno apunta al siguiente. Permiten insertar o borrar elementos fácilmente sin mover el resto.
* Pilas (Stack): Estructura LIFO (Last In, First Out). El último en entrar es el primero en salir (ej. el botón "atrás" del navegador).
* Colas (Queue): Estructura FIFO (First In, First Out). El primero en llegar es el primero en ser atendido (ej. una cola de impresión).
* Árboles: Estructura jerárquica. El más común es el Árbol Binario, donde cada nodo tiene máximo dos hijos. Se usa para búsquedas rápidas.
* Grafos: Conjunto de nodos (vértices) conectados por aristas. Representan relaciones complejas (ej. redes sociales o mapas de carreteras).

## 3. ORGANIZACIONES DE FICHEROS

Un fichero es una colección de registros almacenados en un soporte externo (disco). La "organización" es la forma en que estos registros se disponen físicamente:

* Organización Secuencial: Los registros se escriben uno tras otro. Para leer el registro número 100, hay que pasar por los 99 anteriores. Es eficiente para procesar todos los datos a la vez (ej. nóminas).
* Organización Directa (Aleatoria/Hash): Se utiliza una función matemática (algoritmo de hash) sobre una "llave" del registro para determinar su posición física exacta. Permite acceso inmediato a cualquier dato.
* Organización Secuencial Indexada: Es un híbrido. Los datos están grabados secuencialmente, pero existe un índice (como el de un libro) que permite saltar rápidamente a la zona donde está el dato y luego buscar allí de forma secuencial. Es el modelo base de muchas bases de datos.

Dato clave: En la organización de ficheros, el factor determinante es el tiempo de acceso. La organización indexada es la más equilibrada para sistemas que necesitan tanto listados completos como consultas rápidas de registros sueltos.

Aquí tienes la tabla comparativa entre las dos estructuras de datos lineales más comunes. La clave para diferenciarlas es el orden de entrada y salida:

## Comparativa: Pilas (Stacks) vs. Colas (Queues)

| Característica | Pila (Stack) | Cola (Queue) |
| --- | --- | --- |
| Principio Lógico | LIFO (Last In, First Out) | FIFO (First In, First Out) |
| Concepto | El último en llegar es el primero en salir. | El primero en llegar es el primero en salir. |
| Metáfora | Una pila de platos: solo puedes quitar el que está arriba del todo. | Una cola del cine: el primero que llega es el primero en comprar. |
| Operación de Inserción | Push (se coloca arriba). | Enqueue (se coloca al final). |
| Operación de Extracción | Pop (se extrae del tope). | Dequeue (se extrae del frente). |
| Acceso a datos | Solo se tiene acceso al elemento superior. | Se accede al primero por un lado y al último por otro. |
| Uso común en software | Función "Deshacer" (Ctrl+Z), recursividad, historial del navegador. | Gestión de procesos del SO, colas de impresión, buffers de video. |

## Resumen para tu documento

   1. Pilas: Son ideales para procesos que necesitan "volver atrás". Cuando una función llama a otra, el sistema operativo guarda la dirección en una pila de llamadas.
   2. Colas: Son ideales para la comunicación entre procesos que trabajan a diferentes velocidades. Por ejemplo, un procesador envía documentos a la cola de impresión más rápido de lo que la impresora puede imprimir.
