# FUNDAMENTOS DE PROGRAMACIÓN

## 1. CONCEPTOS BÁSICOS Y ESTRUCTURA DE UN PROGRAMA

Un programa es un conjunto de instrucciones ordenadas para resolver un problema. Su estructura típica incluye:

* Cabecera: Declaración de librerías y módulos.
* Cuerpo: Declaración de variables globales y definición de funciones.
* Programa Principal (Main): El punto de entrada donde comienza la ejecución.

## 2. REPRESENTACIÓN DE TIPOS DE DATOS

Los datos se almacenan en memoria según su naturaleza:

* Enteros (int): Números sin decimales.
* Flotantes/Reales (float/double): Números con decimales.
* Booleanos (bool): Valores lógicos (true / false).
* Caracteres (char) y Cadenas (string): Texto.
* Punteros: Direcciones de memoria donde reside un dato.

## 3. OPERADORES

Permiten manipular los datos:

* Aritméticos: +, -, *, /, % (módulo o resto).
* Relacionales: == (igual), != (distinto), >, <, >=, <=.
* Lógicos: && (AND), || (OR), ! (NOT).

## 4. INSTRUCCIONES CONDICIONALES

Permiten que el programa tome decisiones (flujo de control):

* If / Else: Ejecuta un bloque si se cumple una condición.
* Switch / Case: Evalúa una variable y elige entre múltiples caminos posibles.

## 5. BUCLES Y RECURSIVIDAD

Se utilizan para repetir tareas:

* Bucles (Iteración):
  * For: Se usa cuando sabemos cuántas veces queremos repetir (con contador).
  * While: Se repite mientras se cumpla una condición.
  * Do-While: Asegura que el código se ejecute al menos una vez antes de comprobar la condición.
* Recursividad: Es cuando una función se llama a sí misma. Requiere siempre un "caso base" para evitar un bucle infinito (desbordamiento de pila).

## 6. PROCEDIMIENTOS, FUNCIONES Y PARÁMETROS

Permiten modularizar el código (dividirlo en piezas reutilizables):

* Función: Realiza una tarea y devuelve un valor.
* Procedimiento: Realiza una tarea pero no devuelve nada (en muchos lenguajes se usa void).
* Parámetros: Son los datos que recibe la función para trabajar. Pueden pasarse de dos formas:
  * Por Valor: Se envía una copia del dato (el original no cambia).
  * Por Referencia: Se envía la dirección de memoria (si la función cambia el dato, cambia el original).

Dato clave: La elección entre recursividad o iteración (bucles) depende del problema. La recursividad suele ser más elegante para problemas matemáticos complejos (como el factorial o Fibonacci), pero consume más memoria de la pila (stack).
