Este bloque completa la comparativa de sistemas operativos, cubriendo los tres pilares del escritorio y servidores, además del ecosistema móvil.
------------------------------
## CARACTERÍSTICAS DE LOS SISTEMAS OPERATIVOS

## 1. WINDOWS (Microsoft)
Es el sistema dominante en el escritorio (consumo y empresa).

* Características Técnicas: Kernel híbrido, sistema de archivos NTFS, registro centralizado y compatibilidad masiva con periféricos.
* Funcionales: Interfaz gráfica (GUI) muy intuitiva, gran ecosistema de software (Office, juegos, ERPs) y excelente integración en redes corporativas mediante Active Directory.

## 2. LINUX (Open Source)
No es un sistema único, sino un Kernel sobre el que se crean "distribuciones" (Ubuntu, Debian, Red Hat).

* Características Técnicas: Código abierto, kernel monolítico modular, multitarea real y multiusuario avanzado. Usa sistemas de archivos como EXT4 o XFS.
* Funcionales: Estabilidad extrema (ideal para servidores), alta seguridad (gestión estricta de permisos) y gratuito. Es altamente personalizable.

## 3. UNIX
Es el ancestro de muchos sistemas modernos (incluyendo Linux y macOS).

* Características Técnicas: Sistema propietario (normalmente), diseñado desde el inicio para grandes estaciones de trabajo y mainframes. Muy portable entre diferentes hardware de gama alta.
* Funcionales: Robustez absoluta. Se utiliza en entornos donde el fallo no es una opción (banca, investigación científica, infraestructuras críticas).

## 4. SISTEMAS OPERATIVOS MÓVILES
Diseñados para pantallas táctiles, ahorro de energía y conectividad constante.

* Android (Google):
* Basado en el kernel de Linux.
   * Abierto y altamente personalizable por los fabricantes.
   * Arquitectura basada en aplicaciones que corren en una máquina virtual (ART/Dalvik).
* iOS (Apple):
* Derivado de BSD (Unix).
   * Sistema cerrado y optimizado exclusivamente para el hardware de Apple.
   * Alta seguridad mediante "sandboxing" estricto (las apps no pueden acceder a datos de otras sin permiso).

------------------------------
## GESTIÓN DE PROCESOS Y MEMORIA
El Sistema Operativo actúa como el director de orquesta, decidiendo qué programa recibe atención de la CPU y cuánta memoria puede ocupar.
## 1. GESTIÓN DE PROCESOS
Un proceso es un programa en ejecución. El SO debe gestionar su ciclo de vida:

* Planificador (Scheduler): Es el algoritmo que decide qué proceso usa la CPU en cada momento.
* Round Robin: Reparte el tiempo equitativamente entre todos.
   * Prioridades: Los procesos críticos (como el teclado o el sistema) van antes que los secundarios.
* Estados de un proceso:
1. Nuevo: Se está creando.
   2. Listo: Esperando a que el planificador le dé tiempo de CPU.
   3. Ejecución: Usando la CPU.
   4. Bloqueado: Esperando a que termine una tarea externa (ej. leer un archivo del disco).
   5. Terminado: Ha finalizado su ejecución.
* Multitarea: La sensación de que el PC hace varias cosas a la vez es porque el SO cambia de proceso miles de veces por segundo (conmutación de contexto).

## 2. GESTIÓN DE MEMORIA
El SO debe asegurar que cada proceso tenga su propio espacio y no interfiera con los demás.

* Jerarquía de memoria: El sistema mueve datos entre los registros de la CPU (ultra rápidos), la memoria caché, la RAM y el disco.
* Memoria Virtual: Si la RAM se llena, el SO utiliza una parte del disco duro como si fuera memoria (archivo de paginación en Windows o swap en Linux). Es más lenta, pero evita que el sistema se detenga.
* Paginación y Segmentación:
* Paginación: Divide la memoria en trozos iguales (páginas) para evitar que queden huecos vacíos imposibles de usar (fragmentación).
   * Segmentación: Divide la memoria en trozos según el tipo de datos (código, variables, pila).
* Protección de Memoria: Evita que un error en una aplicación (como Chrome) pueda escribir en el espacio de memoria de otra o del propio Sistema Operativo, provocando el famoso "pantallazo azul".

------------------------------
Dato clave: En el Administrador de Tareas (Windows) o con el comando top (Linux), puedes ver en tiempo real cuánta memoria y CPU consume cada proceso.
