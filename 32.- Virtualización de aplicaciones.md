## VIRTUALIZACIÓN DE ESCRITORIOS Y APLICACIONES

## 1. VIRTUALIZACIÓN DE PUESTOS DE TRABAJO (VDI)
La VDI (Virtual Desktop Infrastructure) consiste en ejecutar el sistema operativo de escritorio (ej. Windows 10/11) dentro de máquinas virtuales alojadas en un servidor central o en la nube. El usuario accede a su escritorio de forma remota desde cualquier dispositivo.

* Ventajas:
* Seguridad: Los datos nunca salen del centro de datos, solo se envían "píxeles".
   * Movilidad: El usuario tiene el mismo escritorio en la oficina, en casa o en una tablet.
   * Gestión centralizada: Actualizar o parchear el sistema operativo se hace una sola vez para todos los usuarios.
   * Aprovechamiento de hardware: Permite usar equipos antiguos (Thin Clients) para ejecutar sistemas modernos.

## 2. VIRTUALIZACIÓN DE APLICACIONES
En lugar de virtualizar todo el escritorio, solo se virtualiza la aplicación individual. La aplicación se ejecuta en el servidor pero aparece en la pantalla del usuario como si estuviera instalada localmente.

* Ventajas:
* Eliminación de conflictos: Permite ejecutar dos versiones diferentes de la misma app (ej. dos versiones de Java) sin que choquen.
   * Despliegue instantáneo: No requiere instalación en el PC del cliente.
   * Aislamiento: Si la aplicación falla, no afecta al sistema operativo local del usuario.

## 3. ESCENARIOS DE APLICACIÓN

* Teletrabajo: Acceso seguro a herramientas corporativas desde redes domésticas.
* Entornos educativos/Laboratorios: Configurar equipos con software específico que se "resetea" tras cada uso.
* Centros de llamadas (Call Centers): Puestos de trabajo estandarizados con alta rotación.
* Seguridad crítica: Manejo de datos bancarios o médicos donde no se permite el almacenamiento local.

## 4. GESTIÓN Y MONITORIZACIÓN
Para que estos entornos funcionen, se requieren herramientas de control que supervisen:

* Broker de conexiones: El software que asigna a cada usuario su escritorio o aplicación disponible.
* Consumo de recursos: Monitorizar la CPU, RAM y latencia de red para asegurar que el usuario no note "lentitud".
* Perfiles de usuario: Gestionar que la configuración personal (fondo de pantalla, archivos, favoritos) se mantenga independientemente de a qué máquina virtual se conecte el usuario.

## 5. SOLUCIONES DE MERCADO
Las principales plataformas líderes son:

* VMware Horizon: Estándar de la industria para VDI robustos.
* Citrix Virtual Apps and Desktops: Líder histórico en virtualización de aplicaciones y entrega remota.
* Microsoft Azure Virtual Desktop (AVD): La solución nativa en la nube de Microsoft.
* Amazon WorkSpaces: Escritorios virtuales como servicio (DaaS) en AWS.

Esta comparativa es clave para entender cuándo compensa invertir en una infraestructura completa o cuándo basta con servir herramientas sueltas.
## VDI (Escritorio Completo) vs. Aplicaciones Publicadas

| Característica | VDI (Virtual Desktop Infrastructure) | Virtualización de Aplicaciones |
|---|---|---|
| Experiencia de usuario | El usuario siente que tiene un "PC completo" remoto. | La aplicación parece una ventana más en su PC local. |
| Consumo de recursos | Alto: Requiere RAM y CPU para todo un Sistema Operativo. | Bajo: Solo consume recursos para ejecutar la aplicación. |
| Control y Personalización | Máximo: El usuario puede (si se le permite) cambiar fondos, iconos y ajustes. | Mínimo: Solo se interactúa con las funciones de la aplicación. |
| Complejidad de Gestión | Alta: Hay que gestionar parches y licencias de cada SO virtual. | Baja: Solo se gestiona y actualiza la aplicación en el servidor. |
| Uso ideal | Usuarios avanzados, desarrolladores o teletrabajo 100%. | Usuarios que solo necesitan una herramienta específica (ej. SAP, Excel). |
| Almacenamiento | Necesita mucho espacio para las imágenes de disco de cada usuario. | Necesita poco espacio; los datos suelen guardarse en red. |

------------------------------
## Resumen de decisión:

* Elige VDI si buscas sustituir por completo el ordenador físico del empleado o si necesitas un aislamiento total por seguridad.
* Elige Aplicaciones Publicadas si el empleado ya tiene un PC potente pero necesita acceder a una herramienta corporativa antigua, pesada o que no es compatible con su sistema local.
