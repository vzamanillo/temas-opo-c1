# SISTEMAS OPERATIVOS: CLIENTE Y SERVIDOR

## 1. SISTEMAS OPERATIVOS CLIENTE (WINDOWS)

Diseñados para la interacción directa con el usuario, priorizando la usabilidad y la compatibilidad de hardware/software.

* Escritorio: Es la interfaz gráfica de usuario (GUI). Incluye la barra de tareas, el menú de inicio y el área donde se gestionan las ventanas.
* Panel de Control y Configuración: Herramientas para la gestión del sistema (usuarios, redes, dispositivos, actualizaciones). Actualmente conviven el Panel de Control clásico y la app de Configuración moderna.
* Gestión de Programas: Windows gestiona las aplicaciones mediante procesos. El Administrador de Tareas permite monitorizar el uso de CPU/RAM y finalizar procesos bloqueados.
* Registro de Windows: Es una base de datos jerárquica centralizada donde el SO guarda todas las configuraciones del sistema, del hardware y de las aplicaciones.
* Nota: Un error en el registro puede inutilizar el sistema. Se edita con el comando regedit. [1, 2, 3, 4, 5]

## 2. SISTEMAS OPERATIVOS SERVIDOR

Diseñados para ofrecer servicios a múltiples clientes simultáneamente, priorizando la estabilidad, seguridad y rendimiento.

* Características:
  * Multiusuario y Multitarea: Capacidad real de gestionar cientos de sesiones.
  * Robustez: Pueden funcionar meses sin reiniciar.
  * Sin GUI obligatoria: Muchos servidores se gestionan por consola (PowerShell o SSH) para ahorrar recursos.
* Elementos Constitutivos: El Kernel (núcleo), el sistema de archivos (NTFS en Windows Server), la pila de protocolos de red y las herramientas de administración remota. [6, 7]

## 3. SERVICIO DE DIRECTORIO (ACTIVE DIRECTORY - AD)

Es el servicio de Microsoft que permite la gestión centralizada de todos los recursos de una red (usuarios, equipos, impresoras, políticas). [8]

* Estructura: Se organiza en Objetos (usuarios), Unidades Organizativas (departamentos), Dominios, Árboles y Bosques.
* GPO (Group Policy Objects): Permiten aplicar reglas de seguridad a miles de equipos a la vez (ej. "impedir el uso de USB" o "cambiar el fondo de pantalla corporativo"). [9, 10, 11, 12]

## 4. IMPLEMENTACIÓN DE DNS EN ACTIVE DIRECTORY

El servicio DNS es obligatorio para que Active Directory funcione. [13]

* Localización de servicios: Los equipos usan DNS para encontrar al Controlador de Dominio (mediante registros SRV).
* Zonas integradas: En Windows Server, las zonas DNS se guardan dentro del propio Active Directory, lo que permite que la base de datos de nombres se replique automáticamente y de forma segura entre todos los servidores del dominio. [14, 15, 16, 17]

Dato clave: Mientras un SO Cliente se apaga al final del día, un SO Servidor está diseñado para la Alta Disponibilidad, permitiendo incluso cambiar piezas de hardware (discos, fuentes) "en caliente" sin detenerse. [18]

Para que el servicio DNS funcione correctamente dentro de un Directorio Activo o en Internet, se utilizan diferentes "etiquetas" o tipos de registros según la información que queramos resolver.
Aquí tienes los más importantes para tu documento:

## Principales Registros DNS

| Tipo de Registro | Nombre | Función |
| --- | --- | --- |
| A | Address | Traduce un nombre de dominio a una dirección IPv4 (Ej: www.google.es → 142.250.184.3). |
| AAAA | IPv6 Address | Igual que el registro A, pero para direcciones IPv6. |
| CNAME | Canonical Name | Crea un alias. Redirige un nombre a otro nombre (Ej: ://web.com → web.com). |
| MX | Mail Exchange | Indica a qué servidor deben enviarse los correos electrónicos del dominio. |
| NS | Name Server | Indica cuáles son los servidores que tienen la autoridad sobre ese dominio. |
| TXT | Text | Permite insertar texto libre. Se usa hoy para seguridad y validación (como evitar que otros suplanten tu correo). |
| SRV | Service | Indica la ubicación de servicios específicos. Es vital en Active Directory para que los PCs encuentren al Controlador de Dominio. |
| SOA | Start of Authority | Contiene información administrativa sobre la zona (número de serie, tiempos de actualización). Solo hay uno por zona. |
| PTR | Pointer | El inverso del registro A. Traduce una IP en un nombre (Resolución Inversa). |

## Importancia en Active Directory

En un entorno de red profesional, el registro SRV es el más crítico. Sin él, los usuarios no podrían iniciar sesión en sus ordenadores porque el equipo no sabría qué servidor es el que valida las contraseñas.

## Resumen de gestión

* Zona Directa: Contiene los registros A, CNAME, MX, etc. (Nombre → IP).
* Zona Inversa: Contiene los registros PTR (IP → Nombre).
