# COMPUTACIÓN EN LA NUBE (CLOUD COMPUTING)

El Cloud Computing es el modelo que permite el acceso bajo demanda, a través de la red, a un fondo compartido de recursos de computación configurables (redes, servidores, almacenamiento, aplicaciones) que pueden reservarse y liberarse rápidamente.

## 1. BASE TECNOLÓGICA

La tecnología fundamental que permite la nube es la Virtualización. Esta permite crear múltiples instancias virtuales (máquinas virtuales) sobre un único hardware físico, optimizando el uso de recursos y permitiendo la escalabilidad inmediata.

## 2. COMPONENTES PRINCIPALES

* Front-end: La interfaz y aplicaciones que ve el cliente (navegador, app móvil).
* Back-end: La infraestructura física (servidores, sistemas de almacenamiento) y el software de gestión (Hypervisor) que administra los recursos.
* Red de comunicación: Normalmente Internet o líneas dedicadas para conectar ambos extremos.

## 3. MODELOS DE SERVICIO (IaaS, PaaS, SaaS)

Se definen según el nivel de control que tiene el usuario frente al proveedor:

* IaaS (Infrastructure as a Service): El proveedor ofrece hardware virtual (servidores, redes, almacenamiento). El usuario gestiona el Sistema Operativo y las aplicaciones.
* Ejemplo: Amazon Web Services (EC2), Microsoft Azure.
* PaaS (Platform as a Service): El proveedor ofrece el hardware y el Sistema Operativo/Entorno de ejecución. El usuario solo se preocupa de desarrollar y desplegar sus aplicaciones.
* Ejemplo: Google App Engine, Heroku.
* SaaS (Software as a Service): El proveedor ofrece la aplicación lista para usar. El usuario no gestiona nada de la infraestructura.
* Ejemplo: Microsoft 365, Google Drive, Salesforce.

## 4. MODELOS DE DESPLIEGUE (TIPOS DE NUBES)

* Nube Pública: Los recursos pertenecen al proveedor y se comparten entre múltiples clientes (multitenencia) a través de Internet.
* Nube Privada: La infraestructura es de uso exclusivo para una sola organización. Puede estar en sus propias instalaciones o gestionada por un tercero. Ofrece mayor control y seguridad.
* Nube Híbrida: Combina nubes públicas y privadas, permitiendo que los datos y aplicaciones se muevan entre ellas según la necesidad (ej. datos sensibles en privada, picos de carga en pública).
* Multicloud: Uso de varios proveedores de nube pública al mismo tiempo para evitar la dependencia de un solo fabricante (Vendor Lock-in).

Dato extra para tu documento: La computación en la nube se rige por el modelo de "pago por uso" y la "elasticidad" (capacidad de aumentar o disminuir recursos automáticamente).

## Resumen para tu documento

   1. IaaS (Infraestructura): Tú alquilas el "ordenador vacío". Tú instalas el Windows/Linux y tus programas. Tienes el máximo control.
   2. PaaS (Plataforma): Tú alquilas un "entorno de trabajo". Te dan el sistema operativo listo para que tú solo subas el código de tu aplicación.
   3. SaaS (Software): Tú alquilas el "servicio final". Solo entras con tu usuario y contraseña y usas el programa. Tienes el mínimo control técnico.
